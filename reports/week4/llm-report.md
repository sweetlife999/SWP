# LLM Usage Report — Week 4

## Tools used

**Claude Code (Claude Sonnet 4.6)** — used throughout Sprint 4 via the `claude` CLI and the VS Code extension.

---

## How AI tools were used

### Report and documentation writing

Claude Code was used to draft and maintain the following files:

- `docs/roadmap.md` — initial structure and sprint goal wording
- `docs/user-stories.md` — migration of user stories from Assignment 3 into GitHub-linked index format
- `reports/week4/reflection.md`, `retrospective.md` — structure and initial content based on actual sprint events

In all cases, the team reviewed, edited, and approved the generated content. Filler text was removed or replaced with project-specific information.

### Debugging

Claude Code helped diagnose bugs:

1. Multiple-choice questions in the Forms Builder could not be edited due to a CSS class collision.
2. Newly-created kanban cards were hidden by the default priority filter because of the automatically enabled filter.

---

## What was done by the team without AI

- Customer interview and sprint review meeting
- All code implementation (frontend components, backend FastAPI routes, database migrations)
- PR reviews and merge decisions
- Repository setup, branch protection rules, and issue template design
